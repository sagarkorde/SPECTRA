# SPECTRA

**Spectral-Probabilistic Ensemble Clustering for Transaction Recognition and Authentication**

SPECTRA is a seven-module pipeline for Bitcoin transaction analysis that
combines spectral graph embeddings, a variational autoencoder (VAE) latent
space, HDBSCAN density clustering, Wasserstein-distance concept-drift
detection, Bayesian trust scoring, Markov-chain behavioral profiling, and an
inductive GraphSAGE classifier into a single, interpretable pipeline.

Unlike tabular classifiers that exploit label-feature entanglement inherent
to rule-annotated Bitcoin datasets, SPECTRA operates on the address-level
transaction graph, so its predictions are temporally fair by construction.

## Architecture

| Module | File | Role |
|---|---|---|
| S — Spectral | `spectra/graph.py` | Directed weighted address graph; spectral embeddings from the normalized graph Laplacian (randomized SVD) |
| E — Entropy | `spectra/features.py` | Mutual-information feature ranking + NMF decomposition |
| C — Cluster | `spectra/vae.py`, `spectra/cluster.py` | VAE latent representation, reconstruction-based anomaly score, HDBSCAN clustering, Wasserstein drift detection |
| P — Trust | `spectra/profiler.py` | Bayesian trust scoring (Beta conjugate posterior) |
| T — Markov | `spectra/profiler.py` | Markov-chain behavioral profiling over cluster states |
| R — GNN | `spectra/classifier.py` | Inductive GraphSAGE transaction-type classifier |
| A — Anomaly | `spectra/pipeline.py` | Fuses VAE / Markov / trust signals into a unified anomaly index |

`spectra/baselines.py` implements the ten comparison baselines (K-Means,
DBSCAN, Isolation Forest, Autoencoder+K-Means, Plain HDBSCAN, XGBoost,
EvolveGCN, BERT4ETH-BTC, Graph Autoencoder, AA-GNN) and `spectra/visualize.py`
generates all figures.

## Installation

Requires Python 3.10+.

```bash
git clone https://github.com/sagarkorde/SPECTRA.git
cd SPECTRA

# PyTorch first, with the CUDA wheel matching your system
# (CPU-only: drop the --index-url line)
pip install torch --index-url https://download.pytorch.org/whl/cu128
pip install torch_geometric

# Everything else
pip install -r requirements.txt
# or, for an editable install of the spectra package itself:
pip install -e .
```

Optional: topological-data-analysis features (`giotto-tda`) are only needed
for the optional persistence-diagram figure:

```bash
pip install giotto-tda
```

## Data

See [`data/DATASET.md`](data/DATASET.md) for the full schema, summary
statistics, and known data-quality caveats.

The dataset itself (5.88M Bitcoin transactions, 53 fields) is distributed
separately via IEEE DataPort:

- DOI: [10.21227/bxmt-mn56](https://doi.org/10.21227/bxmt-mn56)

Download it and set `data.path` in `configs/config.yaml` to point at the
local Parquet file.

## Configuration

All experiment parameters live in [`configs/config.yaml`](configs/config.yaml):
random seed, dataset sampling, feature engineering, and every module's
hyperparameters (spectral dimensions, VAE architecture, HDBSCAN thresholds,
drift window, Bayesian priors, GraphSAGE architecture, baseline
hyperparameters). Copy this file and pass `--config path/to/your.yaml` to
run with a different configuration without touching the defaults.

## Usage

```bash
# Full pipeline (data -> spectral -> VAE/cluster -> trust/Markov -> GNN -> baselines -> ablation -> figures -> results)
python run_spectra.py

# Individual phases (each checkpoints, so you can resume)
python run_spectra.py --phase data        # load data, build graph, spectral embeddings
python run_spectra.py --phase spectra     # + VAE, HDBSCAN, GraphSAGE
python run_spectra.py --phase baselines   # + all 10 baselines
python run_spectra.py --phase ablation    # + 6 ablation variants
python run_spectra.py --phase figures     # + all figures
python run_spectra.py --phase results     # + CSV / LaTeX result tables

# Clear checkpoints and start over
python run_spectra.py --reset

# Use a different config
python run_spectra.py --config configs/my_config.yaml
```

Temporally-fair baseline re-evaluation (removes `block_height`,
`block_time`, `year` before re-selecting features and re-running every
baseline plus SPECTRA):

```bash
python run_fair_baselines.py
```

## Reproducing results

1. Install dependencies (see [Installation](#installation)).
2. Download the dataset and update `configs/config.yaml` → `data.path`
   (see [Data](#data)).
3. `python run_spectra.py` for the full pipeline, or `--phase results` if
   you only need the metrics table.
4. `python run_fair_baselines.py` for the temporally-fair comparison.

Determinism: `configs/config.yaml` fixes `seed: 42`, used to seed NumPy,
PyTorch, and every stochastic baseline. Train/validation/test splits are
70/15/15 and chronological (not shuffled), so no future transaction leaks
into training.

## Testing

```bash
pip install -e ".[dev]"
pytest tests/
```

The current suite is a set of import and config-parsing smoke tests, not a
full correctness suite for the modeling code.

## Reproducibility notes

- **Hardware used for reported runtimes:** Intel Core i9-13th Gen, 64 GB
  DDR5 RAM, NVIDIA RTX 4060 Laptop GPU (8 GB VRAM), Windows 11.
- **Software versions:** Python 3.10, PyTorch 2.7, PyTorch Geometric 2.6,
  scikit-learn 1.4, hdbscan 0.8.38, DuckDB 0.10 (see `requirements.txt` for
  version floors across the full stack).
- **Label provenance:** transaction-type labels are derived from
  deterministic heuristics over structural features
  (`input_count`, `output_count`, value ratios). `run_fair_baselines.py`
  exists specifically because these heuristics can leak into raw tabular
  features; see `data/DATASET.md` for details.
